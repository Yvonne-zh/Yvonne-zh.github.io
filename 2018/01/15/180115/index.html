风险管理与内部控制课程笔记

发表于 2018-01-15 | 分类于 学习
2017-2018第一学期风险管理与控制课程笔记

李心合部分

一、内控背景

  • 政治背景：政治生态变化——反腐败——内控是发腐的基础性工程。
  • 经营环境：宏观经济环境、技术环境、商业模式创新 ——高风险经营时代。
  • 风险事件：国有企业的重大资产损失事件：省属国企的系列事件；上市公司的财务舞弊案件：舜天船舶事件；私营企业的经营失败案件：雨润事件。

二、对“内部控制”的理解

《基本规范》：“内部控制，是由企业董事会、监事会、经理层和全体员工实施的、旨在实现控制目标的过程。”

我的理解：内控是通过流程（“形”）实现制衡（“神或魂”），达到有效防控风险“目标”的过程。

（一）内控之“形”：流程

基本流程或流程内核：提——审——决

  • “提”：提出、申请。
  • “审”：审核、审定。
  • “决”：决定、审批。
  • 附加流程：评审、鉴定、审计等。

上市公司财报签字：

  • 财会记录负责人
  • 财务负责人：总会计师、财务总监等，分管财务的总经理、副总裁（这两种多见），董事长直管财务（少见）
  • 法定代表人：董事长或总经理

流程：编制人编制 -> 财会负责人一审 -> 提交给财务负责人二审 -> （注会的社会审计） -> 提交给董事会审议 -> 法定代表人签字 —> 对外披露
注册会计师在审计报告而非财务报告上签字

  • 流程的合规性：公司法、三重一大制度、监管法规等。
  • 流程的效率性：避免控制过度、协调控制与效率关系。
  • 流程的风险性：避免控制不足、有效防控风险（省属企业分权管理案例）。
  • 流程的有效性：避免程序化和走过场（省属企业案例）。
  • 流程的经济性：避免控制成本过高——流程长、高成本控制工具（先进的平衡计分卡等）或程序（集体决策）。

控制过度和控制不足都要避免
控制过度会造成

  1. 效率低
  2. 成本高
  3. 效果差（节点越多，搭便车现象越严重）
  4. 责任难追究

（1）“提”（申请）

谁提出？

  • 个人（涉及个人事项）提出
  • 机构或部门（公司事项）提出［机构——董事会、总经理办公会等，部门—经办部门或归口部门］

何种方式提出

  • 表（申请表）：内部的
  • 单（报销单等） ：收据发票，是外部来的
  • 报告（申请报告）：注意与管理方案相关

什么时间提出

  • 有时间要求的
  • 无时间要求的

（2）“审”（审核）

谁审

  • 一审（主管或专职审核岗位）
  • 二审（部门负责人）
  • 三审（分管领导）
  • 会审会签

审什么

  • 业务审核（业务的合规性、合理性）
  • 财务审核（形式审核—单证的完整性和完备性，实质审核——单证的真实性、合规性和财务收支的合理性）完整完备、真实合理合规

收据，单位的自制收据不可报销，财政机关颁发的行政事业费收据可以报销

审的时间要求

  • 流程中应当有时间规定

（3）“决”（决定、审定、审批等）

谁有权决定或审批

  • 集体决策（党委会、董事会、办公会、职代会等）
  • 个人审批（董事长、总经理、分管领导、部门负责人）

党委会：中层人事任免（重要干部任免）

董事会：见公司法

  1. 审批预算
  2. 大额长期贷款和大额资金调度
  3. 年度经营计划（单项不一定董事会批）
  4. 任免财务负责人等

总经理办公会：不上报给董事会的重大事项

职代会（不是终极审批，要再提交给董事会审批）

  1. 职工身份（如有终身制变为劳动合同制）
  2. 职工薪酬改革方案
  3. 职工代表（员工董事和员工监事）
  4. 企业改制，特别是产权制度

个人审批的分管领导：总经济师、总工程师、总会计师、总审计师、总法律顾问等

何种形式决定或审批

  • 决议（集体）
  • 审批意见（个人）

审批权限如何设置

  • 法律法规（公司法与三重一大决策制度等）［三重一大一定要集体决策］
  • 公司类型（国企、上市公司与私企等）与公司地位（母公司、子公司与分公司）［母公司的审批作为前置性程序，同意了才能提交子公司董事会（因为董事会批过了就有法律效力了）］
  • 计划或预算（内外或超计划预算）
  • 业务或事项的重要性（定性与定量）

三重一大

  • 重大决策事项
  1. 企业发展战略、破产、改制、兼并重组、资产调整、产权转让、对外投资、利益调配、机构调整等方面的重大决策;
  2. 企业党的建设和安全稳定的重大决策。
  • 重大项目安排事项
  1. 年度投资计划
  2. 融资、担保项目
  3. 期权、期货等金融衍生业务项
  4. 重要设备和技术引进
  5. 采购大宗物资和购买服务
  6. 重大工程建项目。
  • 重要人事任免事项
  1. 中层以上干部和所属单位领导班子成员的任免、聘用、解除聘用和后备人选的确定；
  2. 向控股和参股企业委派股东代表，推荐董事会、监事会成员和经理、财务负责人。
  • 大额度资金运作事项
  1. 年度预算内大额度资金调动和使用;
  2. 超预算的资金调动和使用;
  3. 对外大额捐赠和赞助。

业务审批与财务审批

  • 两位分管领导的重复审批问题
  • 前置性审批对后置性审批权限设置的影响

决的时间要求

  • 流程中应规定时间

（4）附件程序

评审、论证

  • 常设的专门委员会（董事会下属的和总经理办公会下属的专委会）
  • 临时性的专家委员会（前期论证性的和后期验收性的）

鉴定

  • 技术性鉴定（技术先进性、资产报废等）
  • 合规性鉴定（司法鉴定等）
  • 真实性鉴定（身份和单证等）

审计

  • 先审计后提交董事会，一共两个审计报告提交（年报和内控报告）
  • 三种审计，管理审计、业务审计和财务审计
  • 管理审计（管理的工具箱的工具要审计）：战略审计、预算审计（行政事业单位七八月开始编，企业九月开始编）、内控审计、绩效审计等
  • 业务审计：工程审计（全过程跟踪审计）、教学审计、科研审计、安全管理审计、社会服务审计等
  • 财务审计：报表审计和财务收支审计

（5）需要设计哪些流程

全覆盖（内控的特征）
业务类流程：研发、设计、采购、存货、制造、质量、营销、售后服务、工程、外包、招投标、安全、

  • 工程建设招投标

公开招标（一般标的物超过30万就要公开招标）

  1. 找招标代理机构（招投标平台、中介）
  2. 提出代理需求，招标和中标的基本条件（招标需求文件）
  3. 招标代理机构的具体实施：编写具体的招标文件、发标、投标、接标、在规定时间开标、评标、定标、清标
  4. 报给招投标单位，单位进行复核（代理机构违规时有权推翻）

邀请招标

财务类流程：融资、担保、投资、资金、财务报告、资产、成本费用、税收事项等。

管理类流程：战略、组织架构、流程管理、风险评估与报告、合同、法律事务、信息系统、企业文化等。

  • 合同

从法律角度，一般要件（主体、客体、时间、权利与义务等）齐全
从风控角度，着眼于签订、管理的流程

  1. 合同签订：合同尽调、拟定草案、合同谈判、修改合同、相关部门会审会签（至少有业务部门、法务部门和财务部门）、法定代表人（或其授权人）签字
  2. 合同履行
  3. 合同变更
  4. 合同总结（反馈），履行情况如何
  5. 合同分析
  6. 合同考核

人资类流程：人力资源、岗位管理、绩效管理、薪酬管理、工会等。

  • 薪酬管理包括：薪酬方案的制定（要过董事会）和薪酬发放的流程

（二）内控之“神”：制衡

制衡机制：流程式制衡、竞争式制衡、分离式制衡、合作式制衡、审批式制衡、集中式制衡、信息式制衡、监督式制衡、问责式制衡、系统式制衡。

竞争式制衡

  • 外部竞争：市场竞争（招标等）
  • 内部竞争：岗位竞争等［任用要用竞聘制，高管少，中层（部门负责人）多］

流程式制衡

  • 设计制衡性的流程（提审决）

分离式制衡

  • 不相容职务的分离：权能分离（三权分离）和性态分离（三流分离）

权能分离：

  • 治理层：决策（董事会）、执行（总经理办公会）、监督（监事会）
  • 中间层：管理（法务、行政、财务）、经办（业务部门）、监督（内审、纪检监察等）
  • 操作层面：管理、操作、检查［联系具体岗位］

性态分离：钱、账、物

  • 分离方式：机构分设与岗位分设；横向分离与纵向分离
  • 三个角度：流程、功能、性态
  • 设立不相容岗位时考虑成本效益原则和重要性原则

合作式制衡

  • 会审：常用于合同和重要事项决策
  • 会签：一般指文件，文件发布前会签
  • 会议：主要关注董事会、总经理办公会、各种类型的专门委员会议（专委会）、各种专项工作会议（供应、生产、销售等）、职代会、监事会、党委会
  • 共同决策
  • 共同行动

“三重一大”集体决策就是通过会议、通过共同决策实现合作式制衡

审批式制衡

  • 集体审批与个人审批
  • 财务审批与业务审批

集中式制衡

  • 全口径集中，集团式集中：物流(集中采购)、信息流(集中核算)、资金流(财务共享)和人流(集中管理)
  • 功能式集中：归口集中管理、分散经办。

资金的管理方式：

  1. 分散管理：子公司的钱子公司自己管
  2. 集中管理，“资金池”模式，大势所趋
    • 内部成立资金集中/财务共享中心等
    • 成立财务公司（独立法人，过去人行批，现在银监会批）：由集团成立，在集团内部负责资金融通的非银行金融企业（吸收发放存贷款）［钱主要来自集团内部的和同业拆借］
    • 公司银行：金融资本和产业资本结合形成财团，可以向社会吸收发放存贷款（财务公司不能面向社会）

物流的管理方式

  1. 全口径集中，流程：列计划、给总公司申请、总公司批后转给采购中心、采购中心直接配送
    • 要求：信息系统、发达的物流体系、企业的组织形式是分公司而不是子公司
  2. 大宗物资采购集中：两类物资，金额较大价值较高的和批量较大的

信息流的管理方式

  • 三大信息流（三大核算）：业务核算、会计核算、统计核算
  • 会计通过财务共享集中核算，通过集中来实现共享

    集中的事项范围：资金集中、开户集中、核算集中、税务集中、票据集中（发票和银行的结算凭证）、人员集中

信息式制衡

  • 公开透明
  • 信息披露

监督式制衡

  • 内审
  • 监事会
  • 纪检监察
  • 监督式委派：产权代表和财务负责人

委派产权代表：委派董事（股权代表）和监事
委派财务负责人：

  • 人员种类：财务总监；财务负责人、财会机构负责人；一般会计人员委派
  • 薪酬方案多为派驻单位承担，委派单位统一发放
  • 功能类型：
    • 监督型，不参与决策管理（如政府向企业委派）
    • 扮演财务负责人角色（即监督、管理，更常见）
    • 监督+管理型
  • 委派类型
    • 直接委派（母公司人员到子公司去）
    • 身份委派（母公司赋予子公司人员新的身份）

问责式制衡

系统式制衡

  • 信息系统固化流程
  • 控制权力弹性

（三）内控目标：风险防控

风险评估与风险库

  • 整体风险（战略风险、市场风险、运营风险、财务风险、法律风险等）和业务风险
  • 风险等级（三级）
  • 风险评估流程：识别—分析—评价—报告

风险态度与风险应对

  • 风险态度：风险最小化与可接受风险水平
  • 风险应对：策略

内控与风险管理的关系：内控是风险管理的组成部分

（1）风险等级划分

（2）风险评估流程

风险：对目标实现造成不利影响的因素和计算（风险是针对目标而存在的）

信息搜集 -> 信息搜集表

  1. 宏观与微观
  2. 内部与外部
  3. 财务与非财务等

风险识别 -> 风险描述

  1. 风险名称。采用“目标或业务+风险”的描述方式。如财务风险、投资风险等。
  2. 风险描述。对风险发生的各种成因及其表现分类描述。

风险分析

  1. 风险发生可能性分析标准。分极高、高、中、低、极低五个等级；
  2. 风险影响程度分析标准。分极高、高、偏低三个等级。

风险评价 -> 风险评价矩阵

  1. 风险分为重大风险、重要风险和一般风险三个等级，划分的标准是按照风险影响程度和风险发生的可能性的组合。

风险应对

  1. 策略：风险承担、风险控制、风险规避、风险转移、风险转换、风险对冲、风险补偿等。

风险报告 -> 风险库

  1. 风险内容。包括风险名称、风险类型、风险描述等；
  2. 风险等级。包括重大风险、重要风险和一般风险三级；
  3. 风险应对。包括应对策略、主要措施等。

风险预警 -> 预警报告

  1. 预警指标体系的设计

担保不能授权，不管金额大小都要提交董事会和股东会

（四）内控载体

内控载体形式：内控矩阵式、制度+流程图式、图表式

  • 流程：谁做？做什么（控制活动）？依据什么做（输入）？做的结果（输出）？做的时间要求？
  • 制度：(1)覆盖内控要素； (2)突出风险防控主题； (3)把握内部制衡主线。
  • 表单：表格、单证、报告
  • 备注：制度形式：章节条款式、数字排列式、图表式

三、内控流程的分类分析

四流：资金流、业务流或物流、信息流、工具流。

  • 资金流：流入、流出、存量等 -> 财务控制。
  • 业务流或物流：采购、使用、存量 -> 存货和固定资产 -> 业务控制。
  • 信息流：业务信息、财务信息、统计信息 -> 信息系统 -> 会计控制。
  • 工具流：战略、预算、合同、考核等等 -> 管理控制。

战略

  1. 财务是战略的组成部分（财务战略和业务战略）
  2. 会计是战略的工具（战略分析需要会计、战略落地需要会计将战略目标分解）

合同

  1. （部分企业）合同归口财务管
  2. 财务部门参与合同谈判评审签订

绩效考核

  1. 财务本身就是绩效考核的一部分（平衡积分法/kpi等，国务院考核国企关注营业收入、经济增加值和净利润）
  2. 会计报表是考核的重要依据

（一）资金流控制

资金流控制内容：资金流量控制与资金存量控制。

资金管理模式：

  1. 分散管理：账户分散、存款分散、收支分散。
  2. 集中管理
  • 谁集中：母公司。
  • 集中方式：内部资金中心、财务公司与企业银行——资金池模式。
  • 集中内容：开户集中、存款集中、收支集中、核算集中。
  • 集中好处：整合资金、降低成本、增进管控。

资金流入控制

经营资金流入控制：应收账款控制——全程信用管理

  • 事前：信用政策设计、客户选择
  • 信用政策：分松紧两种类型，要素包括：
    • 信用期限
    • 信用额度（总额度&单项额度）
    • 信用条件（担保等）
    • 利息附加条件等
  • 客户选择：考虑
    • 收益（创造价值）

内部控制与风险管理整体框架

内部控制

COSO委员会《内部控制——整体框架》中对内部控制的定义：“内部控制是一个要靠组织的董事会成员、管理层和其他员工去实现的过程，实现这一过程是为了合理地保证：

  1. 经营的效果性和效率性；
  2. 财务报告的可信性；
  3. 对有关的法律和规章制度的遵循性。

内部控制的局限性：当发生以下情况时，内部控制失效

  1. 决策失误
  2. 理解制度有误
  3. 领导违规
  4. 串通舞弊
  5. 成本大于效益

内部控制的目标

  1. 经营目标：资产利用的效率及效果
  2. 财务报告目标：财务报告的可靠性
  3. 合规性目标：企业各项获得符合法律法规

内部控制的要素

  • 控制环境
  • 风险评估
  • 控制活动
  • 信息与沟通
  • 检查与监督（监控）

内部控制的基本假定：两个或两个以上的人同时做一件事，比一个人独自做一件事， 出错和作弊的可能性大大减少。（控制与效率是一对矛盾）

内部控制的基本特征：

  1. 牵制权利
  2. 规范做事
  3. 公开透明
  4. 随时检查

控制环境

  1. 管理层的思想和经营作风
  2. 企业的组织结构
  3. 董事会及其所属委员会
  4. 职权设计
  5. 管理者检查监控的具体
  6. 人事管理制度
  7. 公共关系

内部控制所要解决的主要问题

  • 权
  1. 不相容岗位分离
  2. 重要岗位轮岗
  3. 要害岗位离岗
  4. 分权与授权
  5. 战略->组织->岗位->员工能力->绩效
  • 事
  1. 流程规范
  2. 作业优化
  3. 活动标准
  4. 环节控制点与手续
  5. 能力与制度
  • 通
  1. 信息流、业务流、资金流对接
  2. 内部报告系统与持续改进计划
  3. 独立检查与自我评估
  4. 风险预警与分类管理
  5. 管理层级与管理团队

内部控制关注的主要环节

  1. 充分的职责分工(不相容职务分离)
  1. 资产保管与会计相分离
  2. 业务授权与相关资产保管相分离
  3. 经营责任与记账责任相分离
  4. 电子数据处理系统中系统分析员、编程员、计算 机操作员、程序管理员、数据控制员等这些职责相分离
  1. 业务和活动的适当授权
  1. 一般授权：制定统一遵守的政策，各部门、个人在政策规定范围内批准各项业务
  2. 特别授权：个别业务的一次性授权
  1. 充分的凭证和记录
  1. 预先连续编号
  2. 业务发生时立即编制
  3. 简单明了，易于理解
  4. 一式多联，方便实用，便于控制
  5. 格式有利于正确编制
  6. 设计科目表，便于正确记录和分类控制
  7. 编制制度手册，明确记账手续，便于利用会计记录对经济活动实施控制
  1. 对资产和记录的实物控制
  1. 资产：安全性
  2. 记录：防止被改和被盗、丢失

集权与分权的效益分析

对下级的激励 重复建设成本 - -
  1. 对执行情况的独立检查
  1. 没有一个经常性的检查机制，内部控制很可能随着时间的推移而发生变化
  2. 没有人监督和评价其业绩，雇员就可能忘记或者蓄意不遵守控制措施；也可能会变得粗心大意
  3. 无论控制的质量如何，都可能会出现舞弊行为和蓄意错报。

内部控制自我评价(CSA) ：内部审计发展的新方向

  • 企业不定期或定期的对自己及所属子公司的内部控制系统进行评价
  • CSA是一种自我评估运营程序，把传统的只由内部审计人员从事的内部评价转由公司各部门参与作业的人员亲自评估，把内部控制作为组织所有成员的事。
  • 内部审计人员与被评价单位管理人员组成一个小组， 对本部门内部控制的恰当性和有效性进行评估，出具改进建议的报告，由管理者实施。

内部控制自我评价的基本特征

  • 关注业务流程和控制成效
  • 由职能部门和业务部门共同进行
  • 从内部审计人员到业务具体执行人员全体参与
  • 用系统化的方法开展评价活动

内部控制评价的内容

  1. 高级管理阶层对内部控制的重视程度如何？
  2. 管理阶层是否有正确的目标？这些目标是否切实可行并具有竞争性？
  3. 管理层是否建立有效的信息及报告系统，以供决策之用？
  4. 管理阶层是否已建立持续性的风险评估过程，以辨识及衡量经营风险的大小及可能性？
  5. 管理阶层是否明确界定有关经营目的及风险的责任，使权责人员采取必要的行动？
  6. 管理阶层是否已建立有效的控制制度？这些制度是否可合理保证遵循主管机关及有关法令的要求？
  7. 管理阶层是否及时坦诚地与董事会沟通？沟通的内容是否有关、充分及完整？
  8. 董事会及审计委员会是否把优先工作的焦点摆在公司关键性的经营风险上？
  9. 内部审计的工作范围，就经营风险评估而言，是否适当？

内部控制缺陷

内部控制缺陷的定义：内部控制设计或执行存在不足，使得管理层和公司员工在履行其职责过程中，不能及时预防或发现财务报告的错漏。

以下任一情况出现，都意味着内部控制存在设计上的缺陷

  • 针对某一控制目标，缺乏必要的控制点（控制点）
  • 已有的控制设计不当，以致即使正确的按设计的要求执行了控制程序，也不能始终实现控制目标（控制程序）

以下任一情况出现，都意味着内部控制存在执行上的缺陷：

  • 设计恰当的内部控制未按照设计的要求正确执行（违规、舞弊）
  • 控制执行人没有取得必要的授权或缺乏必要的胜任能力（难以胜任）

重要缺陷是控制缺陷的集合

实质性漏洞是重要缺陷的汇总

对控制缺陷的定性

  • 显著缺陷：导致年报或中期报告中的并非无关紧要的错漏无法被预防或侦测出来的可能性较大
  • 实质漏洞：导致年报或中期报告中的重大错漏无法被预防或侦测出来的可能性较大，严重显著缺陷=实质性漏洞
  • 需要做个别和整体的评估
  • 与财务报告相关的内部控制存在实质性漏洞时，管理层不得作出内部控制有效的声明

显著性缺陷的界定

  • 若公司的内部控制在以下任何一方面存在缺陷， 即被认为是显著缺陷：
  1. 对财务会计制度的选择及使用的控制
  2. 防范欺诈舞弊的程序及控制
  3. 对非经常性及非系统性交易的控制
  4. 对期末财务报表流程的控制
  • 若公司的内部控制发生以下情形，则至少存在显著缺陷，同时也是存在实质性缺陷的一个强烈信号：
  1. 对已发布的财务报表进行修改
  2. 外部审计师在本年度审计中发现的重大误报不是由该公司首先发现
  3. 无效的审计委员会监管
  4. 对财务报表流程可靠性至关重要的内部控制或风险评估失效
  5. 处于高度监管行业中的公司，其合规控制失效
  6. 发现高管层存在任何程度的舞弊
  7. 已经发现并报告的重大控制缺陷在经过合理的时间后并未加以改正，及无效的控制环境。

风险管理

缘起

社会与个体组织目标、个人与组织（企业目标的不一致）

  1. 认知能力不足(出差错)
  2. 负责精神不够(出差错)
  3. 习惯老一套、习以为常(低效率)
  4. 滥用职权谋私利(舞弊)

个体的先天局限

  1. 生命有限
  2. 任期有限
  3. 利益冲突

后果

  • 目标不一致，对组织和社会而言，实现目标的过程就会产生不确定性
  • 对这些不确定性的认知、评价、控制，产生了风险管理
  • 对客观存在的这些不确定性予以控制，使其在社会和组织所接受的框架内发挥作用，产生了内部控制

风险管理

风险管理：针对具体目标和对象

内部控制：给予管理的内在要求和运行机制

内部控制和风险管理实质上是同一种工作的两种称谓， 两者辩证统一，互相依存、相互促进

风险分类：对各种风险按其恶性程度做出分类

  • 红：关键或致命风险
  • 黄：一般或轻微风险
  • 绿：规范或标准状态
    实行分类管理，有效防范和控制

风险识别

  • 外部风险：市场、技术、税收、法律、自然环境、会计制度、政治因素等
  • 内部风险：基础设施（制度、信息系统、软硬件）、人员、增长（并购重组、跨国经营、一元与多元）、流程、技术

风险分析

  1. 风险计量指标
  1. 风险敞口(风险可能带来的最大损失)
  2. 波动性 (风险剧烈程度)
  3. 风险发生的概率
  4. 可损度(风险若发生相对可以确定的损失)
  5. 持续时间(风险存在时间长短)
  6. 相关性(各种风险之间的内在关联性
  7. 可接受的风险损失
  1. 风险计量方法
  1. 概率法：风险价值(VaR)：市场风险、信用风险、操作风险、法律风险
  2. 因素分析法——破产预测
  1. 风险分析的目的
  1. 估计风险的严重性——性质
  2. 评估风险发生的可能性——概率
  3. 拟定对策的可行性——控制

风险持续评估与应对

  1. 风险基本类别评估
  1. 固有风险：指假定不存在相关内部控制时，某一帐户或交易类别单独或连同其他帐户、交易类别生重大错报或漏报的可能性。
  2. 剩余风险：错报、检查检查
  1. 风险应对
  1. 风险回避：放弃某项活动，不干了
  2. 风险降低/分散：组合、对冲
  3. 风险分担/转移：保险与外包
  4. 风险承受：自己承担；认命

风险控制基本方法：批准、授权、验证、调节、业绩评价、资产保管（实物管理）、预算、不相容职务分离

信息控制

  • 一般控制：软件开发与系统维护，安全管理
  • 应用控制：业务流程固化，控制自动实施
  • 信息权配置(授权)
  • 理想模式：九大要点（量化战略、强化预算、简化核算、固化作业、整体集成、全面监控、相机管理、优化行为、保证绩效）

信息整合

  • 传统不足：短视（过去式）、短期（年度）、短见（货币计量）
  • 拓展目标：实时反映、信息集成、知识共享
  • 工具支持：平衡计分卡、风筝分析法

沟通

  • SCM
  • CRM
  • IRM
  • 员工关系管理：薪酬、股份、晋升、忠诚度

反舞弊机制

  • 提高透明度
  • 加强平时沟通
  • 举报与投诉
  • 关键点控制与信息披露
  • 岗位或权力行使的风险管理：不相容岗位分离、轮岗、离岗

检查与监督

  • 检查监督的两种基本方式：持续监控、个别评价
  • 内部控制缺陷评价：自我评估（领导层进行），外部评估（实质性漏洞，超出“风险容量”与“风险容限”及制度规定的事项未被发现，或未被纠正）
  • 检查监督的基本制度：内部审计（定期审计、不定期审计、专项审计）

风险管理整体框架(ERM)

  • 目标：从“三性”到“四性” ，从经济性、可靠性、合法性，增加了战略性
  • 要素：从5个变为8个
  • 概念：从抽象的合理保证到“风险边界”到“风险容忍度”的合理保证

《企业风险管理框架》的新进展：

在《内部控制整体框架》基础上增加了：

  1. 一个观念——“风险组合观”
  2. 一个目标——“战略目标”
  3. 两个概念——“风险偏好”和“风险容忍度”
  4. 三个要素——“目标制定”、“事项识别”和“风险 反应”

风险组合观

  1. 在单独考虑如何实现企业各个目标的过程中，企业风险管 理框架更看重风险因素。除单独考虑各个风险因素之外，更有必要从总体的、组合的角度理解风险。
  2. 企业风险管理要求企业管理者以风险组合的观点看待风险， 对相关的风险进行识别并采取措施使企业所承担的风险在风险偏好的范围内。
  3. 对企业内每个单位而言，其风险可能落在该单位的风险容 忍范围内，但从企业总体来看，总风险可以超过企业总体的风险偏好范围。因此，应从企业总体的风险组合的观点看待风险。

战略目标——扩大了报告目标的范畴

  1. 内部控制框架中的财务报告目标只与公开披露的财务报表的可靠性相关，而企业风险管理框架中 的报告目标范围有很大的扩展，该目标覆盖了企业编制的所有报告。
  2. 企业风险管理框架比内部控制框架增加了一个目标—战略目标。
  3. 该目标的层次比其他三个目标更高。企业的风险管理在应用于实现企业其他三类目标的过程中， 也应用于企业的战略制定阶段。

目标的设定

  • 控制追求的目标：战略实现、经济有效、活动合法、信息可靠
  • 控制依据的目标（风险）：经营活动的边界（风险容量/空间边界，增长、风险、收益）、风险可接受程度（风险容量/数量边界）
  1. 经营目标
    1. 绩效指标：利润、企业价值
    2. 平衡计分卡：使企业利益相关方共赢
    3. 价值管理：企业持续发展与长命百岁
  2. 合规目标
    1. 遵纪：不准—禁止；不以权谋私
    2. 守法：依法行权，不滥用职权
    3. 循规：规范做事，制度面前人人平等
  3. 财务报告(公司信息)目标：可靠性
    1. 存在或发生：事实
    2. 完整：已完成，全部反映，时序正确
    3. 权利与义务：责任得到切实履行
    4. 估价与分摊：可检验(公允)
    5. 列报与披露：重要、分类、可理解

风险意识

  • 目的
  1. 积极主动为公司识别主要的风险
  2. 严肃认真地明白风险发生带来的后果
  3. 风险宣传以使全体或有关员工密切关注
  • 培养风险意识的方法 RISK
  1. 收益(Return)： 收益变动我们能否接受？
  2. 免疫(Immunization)： 使不利因素影响最小化，应该采取什么措施？
  3. 系统(Systems)： 我们有发现并计量风险的制度吗？
  4. 知识(Knowledge)： 我们有无能够有效管理风险的经验和对策？
  • 风险偏好
  • 企业在实现目标的过程中愿意接受的风险数量。一般从定性的角度将风险偏好分为：风险喜好、风险中立和风险厌恶三类。
  • 企业也可以采用定量的方法对风险偏好进行衡量，反映企业的目标、收益与风险之间的关系并进行权衡。
  • 企业风险偏好与企业战略直接相关，企业制定战略时，应考虑将该战略的既定收益与企业的风险偏好结合起来。
  • 不同的战略给企业带来的风险也不同，在企业战略的制定阶段就进行风险管理，就是要帮助企业的管理者在不同战略间选择与企业风险偏好一致的战略。风险偏好的概念是 建立在风险容忍度概念基础上的。
  • 风险容忍度
  • 在企业目标实现的过程中，对差异的可接受程度，是企业在风险偏好的基础上设定的对相关目标实现过程中所出现的差异的可容忍限度。
  • 在确定各目标的风险容忍度时，企业应考虑相关目标的重要性，并将其与企业风险偏好联系起来。
  • 将风险控制在风险容忍度之内能够在更大的程度上保证企业的风险被控制在风险偏好的范围内，也就能够从更高程度上保证企业目标的实现。

增加的三个风险管理要素

  • 目标制定：在风险管理框架中，由于要针对不同的目标分析其相应的风险，因此目标的制定自然就成为风险管理流程的首要步骤，并将其确认为风险管理框架的一部分。
  • 事项识别：企业风险管理和内部控制框架都承认风险来自于企业内、外部各种因素，深入探讨潜在事项的概念
  • 风险反应：企业管理者从总体的角度考虑企业选择的所有风险反应方案组合后对企业的总体影响。

风险容量和风险容限：每项活动都在边界内和数量限定水平下进行

  • 风险容量（风险容许发生的边界）：哪些事情绝对不能做？哪些公众或媒体绝对不能得罪？哪些问题绝对不能有？哪些人员绝对不能用？哪些对手绝对不能忽视？哪些供应商绝对不能敷衍？ 哪些客户绝对不能怠慢？
  • 风险容限（风险发生的可接受程度，各项指标也可以是绝对额也可以是百分比）：收入变动、成本变动、员工流失、盈利变动、工作失误的可容忍度、预算变动等

中国企业的内部控制

  • 控制不足：财产损失、决策失误、制度失败、增加校对时间、股东利益受损
  • 控制过度：官僚作风盛行、降低生产力、增加不必要的复杂性、增加周转时间、增加无价值的工作

新版ERM的变化：风险对战略和绩效的重要性增加，风险管理的发力点提前；突出风险管理对绩效的作用，强调风险管理的价值

风险管理与战略的关系

  1. 外部环境不确定性，应用风险管理的方法优化战略选择
  2. 旧版没有清楚解释风险管理如何应用于战略制定的过程，强调的是战略分解到执行过程的风险， 新版强调的是战略选择环节的风险
  3. 一个战略对应一个风险组合，董事会要比较不同风险组合，权衡利弊
  4. 企业愿景、使命和价值观赋予企 业决策层最初的风险偏好
  5. 风险管理可以纠正战略与愿景、 使命不相符的情况，特别是风险组合与风险偏好的匹配度
